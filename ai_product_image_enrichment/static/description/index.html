AI Product Image Enrichment

The complete catalog imagery platform for multi-brand Odoo shops.

What it does

Three things at once, in one module.

1. Find professional images

Reads manufacturer websites with Claude AI to identify and download real product photography. No per-vendor scraping rules. Adapts to new brands automatically.

2. Remove + normalize

Every main image gets the background removed, trimmed to content, and centered on a uniform 1920×1920 transparent canvas. Shop grid looks like one studio shot the whole catalog.

3. Gallery, intelligently

Lifestyle, in-use, and detail shots become gallery images on the product detail page — where context belongs. Per-category opt-in so consumables don't accumulate clutter.

How it works

Flow A — Normalize existing images

Run on your existing catalog to make every product visually uniform.

  1. Module reads each product's main image
  2. Detects background quality with a fast pixel heuristic + Claude vision for borderline cases
  3. Removes background → transparent PNG
  4. Trims, scales, centers on a uniform 1920×1920 canvas with consistent padding
  5. Writes back to image_1920 — Odoo's standard derivatives auto-update

Result: every product card in /shop shows the product at the same proportional size and position.

Flow B — AI image discovery

Run on products missing imagery, or to add gallery shots.

  1. Sitemap-first discovery — tries manufacturer.com/sitemap.xml directly. Falls back to Brave / Google search if needed.
  2. Robots.txt-respecting fetch with per-domain rate limiting.
  3. Claude classification — sends <img> inventory + product context. Conservative bias against logos, banners, and unrelated images.
  4. Strict brand-domain filter — only accepts images hosted on the manufacturer's own domain or CDN. Falls back gracefully if the brand site is unreachable.
  5. Manual review queue — every candidate goes to a kanban board with original + normalized preview side-by-side. Approve, reject, or apply with one click.

Self-learning recipe cache

After 5 successful AI classifications on a domain, the module asks Claude to generate CSS selectors that distill what it learned. Future products on that domain skip the AI call entirely. Anthropic API cost asymptotes toward zero as your catalog grows.

What's included

  • 6 Odoo models: enrichment job, image candidate, scraping recipe, AI usage log, step log, plus product extension
  • 4 wizards: enrich products (AI), normalize-only, bulk candidate review, preview normalization
  • Dedicated menu with full job, candidate, log, and recipe management
  • Settings panel — canvas size, padding, white-BG threshold, budget cap, alert email, gallery opt-in, brand attribute name
  • ir.cron-based async processing — chunks of 5 products per minute, Postgres advisory lock for safe concurrency
  • Per-category gallery toggle + per-product override — chemicals get one image, scrubbers get five
  • Budget enforcement — monthly USD cap with email alerts at 80% and auto-pause at 100%
  • XML-RPC API for headless integrations
  • Original-image backup + revert button on every product

Anti-footgun defaults

  • Manual review queue is the default — auto-apply is opt-in
  • Confidence threshold defaults to 0.7 (conservative)
  • Monthly Anthropic budget defaults to $50 (configurable)
  • Per-domain rate limit (2s) — robots.txt-compliant always
  • Idempotent jobs — re-running on already-enriched products is a no-op
  • Backup field on every modified product — one-click revert

Technical requirements

  • Odoo 19 (Community or Enterprise)
  • Required Python packages in Odoo venv: requests, beautifulsoup4, lxml, Pillow, numpy, anthropic
  • Optional Python packages (only for self-hosted background removal): rembg, onnxruntime
  • API keys: Anthropic + a search provider (Brave free tier works); Photoroom API key optional

Self-hosted Odoo: install via SSH + pip.
Odoo.sh: add to your repo's requirements.txt — auto-installed on deploy.
CloudPepper / managed hosts: contact your provider's support to add the packages — most handle this routinely.
